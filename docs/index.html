Description

This workshop is designed for beginners. We will introduce participants to tools useful in debugging computer programs. We will use the command line tool GDB and provide interactive examples to get you started. We will also introduce you to memory debugging with Valgrind and discuss options for debugging computer programs that utilize parallel processing.

Instructors

  • Pooja Rao (email: pooja.rao@stonybrook.edu)
  • Jeremy Melvin (email: jmelvin@ices.utexas.edu)

Requirements

  1. To install the required software, please follow the instructions at the bottom of the page.
  2. Some prior knowledge of coding with C/C++/Fortran is recommended but not required.

Learning Goals

  1. Familiarity with GDB and its basic commands
  2. Ability to use a debugger to investigate basic issues in a computer program
  3. How to use Valgrind to investigate memory leaks
  4. Potential approaches to debugging programs built for parallel architecture

Topics Covered

  • Basic commands with GDB
  • Memory debugging with Valgrind
  • Additional commands and features of GDB
  • Approaches to parallel debugging
